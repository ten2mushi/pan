# 12: Block Implementation Patterns and Real Examples (with Verbatim Code)

This document shows **how to implement blocks** and how the existing library does it, by copying large relevant sections verbatim. Patterns for sources, generic signal processors, composites, hardware I/O, sinks, and raw mode are covered.

## 1. The Universal Block Template (from BLOCKS_API.md + block.zig)

Minimal signal processing block:

```zig
const std = @import("std");
const radio = @import("radio");

pub const MyBlock = struct {
    block: radio.Block,
    // your state
    gain: f32,

    pub fn init(gain: f32) MyBlock {
        return .{ .block = radio.Block.init(@This()), .gain = gain };
    }

    pub fn process(self: *MyBlock, input: []const f32, output: []f32) !radio.ProcessResult {
        for (input, 0..) |sample, i| {
            output[i] = sample * self.gain;
        }
        return radio.ProcessResult.init(&[1]usize{input.len}, &[1]usize{input.len});
    }
};
```

The `radio.Block.init(@This())` does the comptime magic (see doc 03).

## 2. Generic Filter Blocks (LowpassFilterBlock + FIR — Verbatim)

This is the canonical "comptime generic with N taps" pattern used throughout the signal library.

From `src/blocks/signal/lowpassfilter.zig`:

```zig
// @block LowpassFilterBlock
// @description Filter a complex or real valued signal with a real-valued FIR
// low-pass filter generated by the window design method.
//
// $$ y[n] = (x * h_{lpf})[n] $$
//
// @category Filtering
// @ctparam T type Complex(f32), f32
// @ctparam N comptime_int Number of taps (filter length)
// @param cutoff f32 Cutoff frequency in Hz
// @param options Options Additional options:
//      * `nyquist` (`?f32`, alternate Nyquist frequency in Hz)
//      * `window` (`WindowFunction`, window function, default Hamming)
// @signature in1:T > out1:T
// @usage
// var filter = radio.blocks.LowpassFilterBlock(std.math.Complex(f32), 128).init(10e3, .{});

pub fn LowpassFilterBlock(comptime T: type, comptime N: comptime_int) type {
    return struct {
        const Self = @This();

        pub const Options = struct {
            nyquist: ?f32 = null,
            window: WindowFunction = WindowFunction.Hamming,
        };

        block: Block,
        cutoff: f32,
        options: Options,
        filter: FIRFilter(T, f32),

        pub fn init(cutoff: f32, options: Options) Self {
            return .{ .block = Block.init(@This()), .cutoff = cutoff, .options = options, .filter = FIRFilter(T, f32).init() };
        }

        pub fn initialize(self: *Self, allocator: std.mem.Allocator) !void {
            const nyquist = self.options.nyquist orelse (self.block.getRate(f32) / 2);
            const taps = firwinLowpass(N, self.cutoff / nyquist, self.options.window);
            return self.filter.initialize(allocator, taps[0..]);
        }

        pub fn deinitialize(self: *Self, allocator: std.mem.Allocator) void {
            self.filter.deinitialize(allocator);
        }

        pub fn process(self: *Self, x: []const T, y: []T) !ProcessResult {
            return self.filter.process(x, y);
        }

        pub fn setCutoff(self: *Self, cutoff: f32) !void {
            self.cutoff = cutoff;
            const nyquist = self.options.nyquist orelse (self.block.getRate(f32) / 2);
            const taps = firwinLowpass(N, self.cutoff / nyquist, self.options.window);
            try self.filter.updateTaps(taps[0..]);
        }

        pub fn reset(self: *Self) void {
            self.filter.reset();
        }
    };
}
```

**Key patterns here:**
- The outer function is a comptime type constructor: `LowpassFilterBlock(T, N)`.
- State includes a `block: Block` (the framework handle) + domain state (`filter: FIRFilter`).
- `initialize` uses `self.block.getRate(f32)` — the rate has already been propagated.
- `process` is a thin wrapper; the real work (and the typed signature) lives in the reusable FIRFilter.
- There is a runtime `setCutoff` that can redesign taps on a live graph (via flowgraph.call if you want to do it from outside the block's thread).

Many other filters (highpass, bandpass, bandstop, single-pole, FM deemphasis, matched filter) follow the same skeleton, some using IIR instead of FIR.

## 3. FIRFilter Internals (Core DSP Primitive)

`FIRFilter(T, U)` (from `src/blocks/signal/firfilter.zig`) is **not** a single tapped-delay-line implementation. It is a tagged union that dispatches to one of three backends, selected at `initialize` time based on which optional libraries are loaded:

```zig
pub fn FIRFilter(comptime T: type, comptime U: type) type {
    if (!((T == std.math.Complex(f32) and U == std.math.Complex(f32)) or
        (T == std.math.Complex(f32) and U == f32) or
        (T == f32 and U == f32))) @compileError("Data types combination not supported");

    return struct {
        const Self = @This();

        impl: union(enum) {
            none,
            volk: _FIRFilterBlockVolkImpl(T, U, Self),
            liquid: _FIRFilterBlockLiquidImpl(T, U, Self),
            zig: _FIRFilterBlockZigImpl(T, U, Self),
        } = .none,

        pub fn init() Self {
            return .{};
        }

        pub fn initialize(self: *Self, allocator: std.mem.Allocator, taps: []const U) !void {
            if (platform.libs.volk != null) {
                self.impl = .{ .volk = _FIRFilterBlockVolkImpl(T, U, Self){ .parent = self } };
            } else if (platform.libs.liquid != null) {
                self.impl = .{ .liquid = _FIRFilterBlockLiquidImpl(T, U, Self){ .parent = self } };
            } else {
                self.impl = .{ .zig = _FIRFilterBlockZigImpl(T, U, Self){ .parent = self } };
            }

            switch (self.impl) {
                .none => unreachable,
                inline else => |*impl| try impl.initialize(allocator, taps),
            }
        }

        // deinitialize / process / updateTaps / reset all switch on self.impl
        // and forward to the active backend (`.none => unreachable`).
    };
}
```

**Key points:**
- The type combinations are restricted at comptime to `(Complex,Complex)`, `(Complex,f32)`, `(f32,f32)`; anything else is a `@compileError`.
- Backend priority is volk > liquid > pure-Zig fallback, chosen from `platform.libs`.
- The pure-Zig and volk backends keep state in a growable `state` list (length `taps.len - 1 + x.len`), not a fixed length-N delay line: each `process` shifts the trailing `taps.len - 1` history samples to the front, resizes, copies the new input in, and computes one dot product per output sample. Taps are stored reversed. `reset()` zeroes the state.
- `liquid` delegates to libliquid's `firfilt_*` objects.

## 4. Hardware Source: RtlSdrSource (DynLib + Sync Read — Verbatim Excerpts)

This shows the "optional hardware, dynamic load at initialize time" pattern used for airspyhf too.

From `src/blocks/sources/rtlsdr.zig`:

```zig
const struct_rtlsdr_dev = opaque {};
const rtlsdr_dev_t = struct_rtlsdr_dev;

var rtlsdr_open: *const fn (...) callconv(.c) c_int = undefined;
// ... many more function pointers declared as var at module scope
var rtlsdr_loaded: bool = false;

pub const RtlSdrSource = struct {
    pub const DirectSamplingMode = enum { I, Q };

    pub const Options = struct { bias_tee: bool = false, ... device_index: usize = 0, debug: bool = false, ... };

    pub const RtlSdrError = error{ InitializationError, ReadError };

    const MIN_BLOCK_SIZE = 8192;

    block: Block,
    frequency: f64,
    rate: f64,
    options: Options,
    buf: []u8 = undefined,
    dev: ?*rtlsdr_dev_t = null,

    pub fn init(frequency: f64, rate: f64, options: Options) RtlSdrSource {
        return .{ .block = Block.init(@This()), .frequency = frequency, .rate = rate, .options = options };
    }

    pub fn setRate(self: *RtlSdrSource, _: f64) !f64 { return self.rate; }

    pub fn initialize(self: *RtlSdrSource, allocator: std.mem.Allocator) !void {
        if (!rtlsdr_loaded) {
            var lib = try std.DynLib.open("librtlsdr.so");
            rtlsdr_open = lib.lookup(...) orelse return error.LookupFail;
            // ... lookup ~20 functions, set rtlsdr_loaded = true
        }

        // resolve device index (by serial or numeric)
        const device_index: u32 = ... 

        var ret = rtlsdr_open(&self.dev, device_index);
        if (ret != 0) { ... return RtlSdrError.InitializationError; }

        if (self.options.debug) { ... print strings, gains, etc. }

        // configure bias tee, direct sampling, sample rate, center freq, bandwidth, gain mode, ppm, etc.
        ret = rtlsdr_set_sample_rate(self.dev, @intFromFloat(self.rate));
        ...
        ret = rtlsdr_reset_buffer(self.dev);
        ...

        // Allocate read buffer
        self.buf = try allocator.alloc(u8, 16 * 2 * MIN_BLOCK_SIZE);
    }

    // deinitialize turns off the bias tee (if enabled), calls rtlsdr_close(dev), and frees buf.
```

The actual `process` (verbatim):

```zig
pub fn process(self: *RtlSdrSource, z: []std.math.Complex(f32)) !ProcessResult {
    // Compute minimum read length
    const len = @min(self.buf.len, z.len & ~@as(usize, MIN_BLOCK_SIZE - 1));

    // Check read length is non-zero (i.e. there is sufficient space in output buffer)
    if (len == 0) {
        return ProcessResult.init(&[0]usize{}, &[1]usize{0});
    }

    // Read samples
    var num_read: c_int = 0;
    const ret = rtlsdr_read_sync(self.dev, self.buf.ptr, @intCast(len), &num_read);
    if (ret != 0) {
        std.debug.print("rtlsdr_read_sync(): {d}\n", .{ret});
        return RtlSdrError.ReadError;
    }

    // Convert complex u8 samples to complex float samples
    const num_samples: usize = @divExact(@as(usize, @intCast(num_read)), 2);
    for (0..num_samples) |i| {
        z[i] = std.math.Complex(f32).init((@as(f32, @floatFromInt(self.buf[2 * i])) - 127.5) * (1.0 / 127.5), (@as(f32, @floatFromInt(self.buf[2 * i + 1])) - 127.5) * (1.0 / 127.5));
    }

    return ProcessResult.init(&[0]usize{}, &[1]usize{num_samples});
}
```

**Notable:**
- All heavy lifting (lib load, device open, config) is in `initialize`, not init().
- Uses an internal `buf` allocated in initialize (`16 * 2 * MIN_BLOCK_SIZE` bytes) for the sync read (the librtlsdr API is byte-oriented).
- The read length is clamped to the buffer size and rounded down to a multiple of `MIN_BLOCK_SIZE` via `z.len & ~(MIN_BLOCK_SIZE - 1)`; if the output slice can't hold a full block, it produces 0 samples.
- Each IQ byte pair is converted to a `Complex(f32)` via `(b - 127.5) * (1.0 / 127.5)`, centering the unsigned 8-bit samples around zero and scaling to roughly -1..1.
- `num_samples` (half of `num_read`) is reported as produced; the source consumes nothing (empty consumed array).
- On error, `process` returns `RtlSdrError.ReadError`, which the runner records before unwinding the graph.

This pattern (DynLib open + lookup table of C fns + initialize-time open/config) is reusable for any external SDR or audio lib.

## 5. Composite Blocks (TunerBlock — Hierarchical Wiring — Verbatim)

Composites look like normal blocks to the outside but wire a sub-graph internally.

From `src/blocks/composites/tuner.zig`:

```zig
pub const TunerBlock = struct {
    block: radio.CompositeBlock,
    translator: radio.blocks.FrequencyTranslatorBlock,
    filter: radio.blocks.LowpassFilterBlock(std.math.Complex(f32), 64),
    downsampler: radio.blocks.DownsamplerBlock(std.math.Complex(f32)),

    pub fn init(offset: f32, bandwidth: f32, factor: usize) TunerBlock {
        return .{
            .block = radio.CompositeBlock.init(@This(), &.{"in1"}, &.{"out1"}),
            .translator = radio.blocks.FrequencyTranslatorBlock.init(offset),
            .filter = radio.blocks.LowpassFilterBlock(std.math.Complex(f32), 64).init(bandwidth / 2, .{}),
            .downsampler = radio.blocks.DownsamplerBlock(std.math.Complex(f32)).init(factor),
        };
    }

    pub fn connect(self: *TunerBlock, flowgraph: *radio.Flowgraph) !void {
        try flowgraph.connect(&self.translator.block, &self.filter.block);
        try flowgraph.connect(&self.filter.block, &self.downsampler.block);

        try flowgraph.alias(&self.block, "in1", &self.translator.block, "in1");
        try flowgraph.alias(&self.block, "out1", &self.downsampler.block, "out1");
    }
};
```

Usage in a top-level graph:

```zig
var tuner = radio.blocks.TunerBlock.init(-250e3, 200e3, 4);
...
try top.connect(&source.block, &tuner.block);
try top.connect(&tuner.block, &demod.block);
```

The Flowgraph's alias machinery (see doc 09) recursively flattens this so the rings are created only between the leaf Blocks; the composite is just a wiring and naming facade.

Other composites (WBFM stereo/mono, AM envelope/synchronous, NBFM) are larger versions of the same idea — they hide a whole demod chain behind one "super block".

## 6. Pure Signal Sources (SignalSource, ZeroSource)

These are simple state machines in process().

`SignalSource` (from `src/blocks/sources/signal.zig`) holds a phase accumulator `phase: f32` and an angular increment `omega: f32`. In `initialize` it selects a per-waveform `process_fn` (`.Cosine`, `.Sine`, `.Square`, `.Triangle`, `.Sawtooth`, `.Constant`), seeds `phase` from `options.phase`, and computes `omega = 2 * std.math.pi * (self.frequency / self.block.getRate(f32))`. The chosen `_processX` fills the output slice, advancing `self.phase += self.omega` per sample and wrapping at `2π`. For example, the cosine path:

```zig
pub fn _processCosine(self: *SignalSource, z: []f32) !ProcessResult {
    for (z) |*e| {
        e.* = std.math.cos(self.phase) * self.options.amplitude + self.options.offset;
        self.phase += self.omega;
        self.phase = if (self.phase >= 2 * std.math.pi) self.phase - 2 * std.math.pi else self.phase;
    }

    return ProcessResult.init(&[0]usize{}, &[1]usize{z.len});
}
```

`ZeroSource` (from `src/blocks/sources/zero.zig`) is even simpler — its `process` does **not** memset; it just reports the whole output slice as produced, relying on the fact that ring/output buffers are zero-initialized:

```zig
pub fn process(_: *Self, z: []T) !ProcessResult {
    return ProcessResult.init(&[0]usize{}, &[1]usize{z.len});
}
```

Because they are sources, they must implement `setRate` returning their configured rate, and they drive the whole graph.

## 7. Sinks (BenchmarkSink as Example of Side-Effect + Periodic Reporting)

From `src/blocks/sinks/benchmark.zig` (full, because it's small and illustrative):

```zig
pub fn BenchmarkSink(comptime T: type) type {
    return struct {
        const Self = @This();
        pub const Options = struct { title: []const u8 = "BenchmarkSink", report_period_ms: usize = 3000 };

        block: Block,
        options: Options,
        count: usize = 0,
        tic_ms: u64 = 0,

        pub fn init(options: Options) Self {
            return .{ .block = Block.init(@This()), .options = options };
        }

        pub fn initialize(self: *Self, _: std.mem.Allocator) !void {
            self.count = 0;
            self.tic_ms = @as(u64, @intCast(std.time.milliTimestamp()));
        }

        pub fn deinitialize(self: *Self, _: std.mem.Allocator) void {
            const toc_ms = @as(u64, @intCast(std.time.milliTimestamp()));
            self.report(toc_ms);
        }

        fn normalize(amount: f32) struct { f32, []const u8 } {
            if (amount > 1e9) {
                return .{ amount / 1e9, "G" };
            } else if (amount > 1e6) {
                return .{ amount / 1e6, "M" };
            } else if (amount > 1e3) {
                return .{ amount / 1e3, "K" };
            } else {
                return .{ amount, "" };
            }
        }

        fn report(self: *Self, toc_ms: u64) void {
            // Compute rate
            const sps = @as(f32, @floatFromInt(1000 * self.count)) / @as(f32, @floatFromInt(toc_ms - self.tic_ms));
            const bps = sps * @sizeOf(T);

            // Normalize rates with unit prefix
            const normalized_sps = normalize(sps);
            const normalized_bps = normalize(bps);

            // Print report string
            std.debug.print("[{s}] {d:.2} {s}S/s ({d:.2} {s}B/s)\n", .{ self.options.title, normalized_sps[0], normalized_sps[1], normalized_bps[0], normalized_bps[1] });

            // Reset tic and count
            self.tic_ms = toc_ms;
            self.count = 0;
        }

        pub fn process(self: *Self, x: []const T) !ProcessResult {
            self.count += x.len;

            const toc_ms = @as(u64, @intCast(std.time.milliTimestamp()));
            if (toc_ms - self.tic_ms > self.options.report_period_ms) {
                self.report(toc_ms);
            }

            return ProcessResult.init(&[1]usize{x.len}, &[0]usize{});
        }
    };
}
```

Sinks are where side effects (sound card, file write, stdout, network, visualization) happen. They usually consume all available input each time they are scheduled.

`PulseAudioSink` (`src/blocks/sinks/pulseaudio.zig`) has a similar structure but, like `RtlSdrSource`, dynamically loads `libpulse-simple` via `std.DynLib` and looks up the C functions (`pa_simple_new`, `pa_simple_write`, `pa_simple_free`). It opens the connection in `initialize`, writes samples synchronously with `pa_simple_write` inside `process`, and frees the connection in `deinitialize`. It is an ordinary `process`-based sink, not a raw-mode block.

## 8. Raw Mode Example (from flowgraph tests)

A raw-mode block opts in via `Block.initRaw(@This(), input_types, output_types)` and implements `start(self, SampleMux)` / `stop(self)` instead of `process`. The `TestRawBlock` in `src/core/block.zig` and `src/core/runner.zig` is the minimal illustration of this interface — its `init` calls `Block.initRaw(...)`, and `start`/`stop` just record that they ran:

```zig
pub fn init() TestRawBlock {
    return .{ .block = Block.initRaw(@This(), &[1]type{f32}, &[2]type{ u32, u16 }) };
}

pub fn start(self: *TestRawBlock, _: SampleMux) !void {
    self.started = true;
}

pub fn stop(self: *TestRawBlock) void {
    self.stopped = true;
}
```

In a real raw block, `start` would retain the received `SampleMux` and then (from another thread or callback) call its `getOutputBuffer` / `updateOutputBuffer` (and the input-side equivalents) directly. This is how you integrate a block that is "event driven from the outside" while still participating in the flowgraph's ring ecosystem and termination.

## 9. Common Implementation Gotchas / Best Practices (Synthesized from Code)

- Always store `block: radio.Block` (or CompositeBlock) as the first (or prominent) field so `@fieldParentPtr("block", ...)` works in the wrappers.
- For generic blocks, put the comptime params on the outer fn that returns the struct type.
- Do heavy init (filter design, device open, large allocs) in `initialize`, not in the plain `init` constructor (constructors should be cheap and infallible).
- Use `self.block.getRate(fXX)` inside initialize when you need fs for calculations.
- Return the exact number of samples you consumed/produced in ProcessResult. The mux uses these numbers (scaled by sizeof) to advance the rings.
- For rate-changing blocks, make sure setRate and the actual consume/produce ratio in process agree.
- For stateful filters (FIR/IIR/PLL/AGC), implement `reset()` if you want users to be able to flush history without destroying the block.
- Support live parameter changes (setCutoff, setGain, etc.) by updating internal state and (if needed) reallocating or updating accel plans. These methods are safe to call via flowgraph.call from another thread.
- Test with the BlockTester / BlockFixture from radio.testing (they use the TestSampleMux).

## 10. Vectors/ for Regression Testing

Every block has a corresponding test that runs it against pre-generated input/output vectors in `src/vectors/blocks/...` (generated by `generate.py` using NumPy/SciPy reference implementations).

This is how numerical fidelity is maintained across pure-Zig, liquid, volk, fftw paths, and across refactors.

Example test (lowpass, verbatim from `lowpassfilter.zig`):

```zig
const vectors = @import("../../vectors/blocks/signal/lowpassfilter.zig");

test "LowpassFilterBlock" {
    // 128 taps, 0.2 cutoff, ComplexFloat32
    {
        var block = LowpassFilterBlock(std.math.Complex(f32), 128).init(0.2, .{});
        var tester = try BlockTester(&[1]type{std.math.Complex(f32)}, &[1]type{std.math.Complex(f32)}).init(&block.block, 1e-6);
        try tester.check(2, .{&vectors.input_complexfloat32}, .{&vectors.output_taps_128_cutoff_0_2_complexfloat32}, .{});
    }
    // ... more cases for different cutoff, nyquist, windows, and the f32 type
}
```

Note `BlockTester` is parameterized by arrays of input/output types and constructed with `.init(&block.block, tolerance)`; `tester.check(rate, inputs, expected_outputs, options)` runs the block against the vectors. There is also a `BlockFixture` used by the "change cutoff" test to drive `setCutoff` mid-stream.

This is an excellent pattern for DSP: the framework + the block author get a strong oracle for "did my implementation match the mathematical definition?"

## Conclusion

Block authoring in ZigRadio is intentionally "boring" and "local": one struct, one init, optional initialize/deinit/setRate, one process (or start). All the hard distributed systems / realtime / threading / memory stuff is handled by the four core pieces (Flowgraph, RingBuffer, SampleMux, Runner). This separation is what lets the project have a large, high-quality block library while keeping the core tiny and auditable.

The verbatim examples above (especially the generic filter, the RTL-SDR dynlib source, and the composite) are the templates you should copy when adding new functionality.
